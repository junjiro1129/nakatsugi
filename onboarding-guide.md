# ナカツギ オンボーディング手順書

手動（コンシェルジュ型）でローンチするための手順書。**Part 1（実装）** はローンチ前に一度だけ、**Part 2（運用）** は申し込みごとに繰り返す。

**凡例**　`- [ ]` 未了 ／ `- [x]` 完了 ／ ⏳ 当局・弁護士の回答待ち（設計は確定・文言や方式は後で差し込む）

**方針**　オンボーディングの中身（事業者性確認・本人確認・反社・記録）は特商法／犯収法の回答に関わらず同じなので、待たずに作る。回答で変わるのは「出品者が同意する表記主体の文言（特商法）」と「本人確認を法定方式に引き上げるか（犯収法）」の2点だけ。そこだけ⏳にしてある。

## Part 1　実装（ローンチ前に一度だけ）

### 第0段階　申し込み受付フォームを動かす　← いまここ
LPの見た目はできているが、フォームの送信先が未設定（`action="https://formspree.io/f/XXXXXXXX"`）。ここを実働させる。

- [ ] Formspreeにアカウントを作成（nakahomes.comと同じ要領）
- [ ] 新規フォームを作成し、エンドポイント（`https://formspree.io/f/xxxxxxxx`）を取得
- [ ] `index.html` の `action` の `XXXXXXXX` を実IDに差し替え
- [ ] 通知先メールアドレスを設定（申し込みが届くアドレス）
- [ ] テスト送信 → 初回はFormspreeの確認メールでフォームを有効化
- [ ] AJAX成功パスの確認（ページ遷移せず「送信しました」カードが出る）
- [ ] 通知メールが届くこと／迷惑メールに入らないことを確認
- [ ] ハニーポット（`_gotcha`）でスパムが弾かれることを確認
- [ ] Cloudflare Pagesへデプロイし、独自ドメインで表示
- [ ] フッターの法務リンク（特商法・利用規約・プライバシー）が全ページ相互に飛ぶか確認

### 第1段階　事業者性確認の準備（今回の一番の肝）
金融庁回答の「受取人＝事業者」を満たす証跡をつくる、御社にしかできない確認。

- [ ] 有効な証憑を確定：①適格請求書発行事業者の登録番号 ②開業届の控え ③直近の確定申告書の控え（いずれか一つ）
- [ ] ①インボイス番号のネット照会手順を用意（国税庁 適格請求書発行事業者公表サイトで有効性を確認し、公表される本名を申込者と照合）
- [ ] ②③の目視確認基準を用意（収受日付印またはe-Tax受信通知があること）
- [ ] マイナンバーのマスキング案内文を用意（控えに記載がある場合は該当部分を隠して提出してもらう。御社はマイナンバーを取得しない）
- [ ] 未開業者向けの案内文を用意（e-Taxで開業届を提出→受信通知を提出してもらう＝ビギナーの入口）
- [ ] 却下・保留の基準を決める（証憑が揃わない／本名が一致しない場合）

### 第2段階　本人確認（KYC）の準備
二重確認を避け、Stripe側の審査に乗るのが基本。

- [ ] Stripe Connectの連結アカウント審査が本人確認を担う前提を確認
- [ ] 御社側の追加確認範囲を決める（①の本名照合＋簡易な身元確認まで）
- ⏳ 犯収法の特定事業者該当性の回答次第で、法定方式（ICチップ読取系／令和7年6月改正後）への引き上げ要否を確定

### 第3段階　反社チェックの準備
- [ ] 確認手段と記録方法を決める（表明保証＋確認手順）

### 第4段階　Stripe Connectの準備
- [ ] Stripe Connectアカウントを開設・本人確認
- [ ] 出品者への招待フロー（連結アカウント作成リンク）を準備
- ⏳ チャージモデル（separate charges and transfers）の確定は決済実装フェーズで。照会の前提（当社は資金を保有しない）と一致するかを弁護士＋Stripeで確認

### 記録台帳の準備
- [ ] 記録項目を確定（下記「記録台帳の項目」）
- [ ] 台帳を用意（単一HTMLの記録ツール推奨。これ自体が「事業者に限っている」ことの証拠になる）

### 回答待ちで差し込む項目（プレースホルダ）
- ⏳ 特商法の回答 → 表記主体の同意文言を確定し、規約・オンボーディングに差し込む（本命＝当社が表記主体／代替＝バーチャル住所方式・当社が販売主体）
- ⏳ 犯収法の回答 → 本人確認方式の最終確定

## Part 2　運用（申し込みごとに繰り返すランブック）

1件ごとに、上から順にチェックする。

- [ ] **0. 受付**　フォーム／メールで申し込みを受領。立場・事業内容を確認
- [ ] **1. 事業者性確認**　①②③のいずれかを取得・確認（①はネット照会＋本名照合／②③は目視＋マスキング確認）。未開業なら開業届を案内し、取得後に再開
- [ ] **2. 本人確認**　Stripe Connectへ招待 → 連結アカウント審査を通す（＋①の本名照合）
- [ ] **3. 反社チェック**　実施・記録
- [ ] **4. Stripe接続確認**　連結アカウントが有効化されたことを確認
- [ ] **5. 記録**　記録台帳に記入（誰を・どの証憑で・いつ・誰が確認したか）
- [ ] **6. 利用開始の連絡**　出品者へ開始案内

### 記録台帳の項目（1件＝1行）
- 事業者名／屋号・氏名
- 連絡先（メール等）
- 事業者性の証憑種別（①インボイス／②開業届／③確定申告）と確認日・確認者
- 本人確認の方法・結果
- 反社チェックの結果・日付
- Stripe Connect 接続状況
- 備考（保留・却下理由など）

## メモ
- この手順書は、ローンチ進捗チェックリスト（launch-checklist）のC系（オンボーディング）を実作業レベルに落としたもの。
- 第0〜第1・第3〜第5は回答を待たずに完成させられる。⏳の2点（特商法の表記主体文言／犯収法のKYC方式）だけが回答待ち。
- 事業者性の記録は、金融庁回答の前提を満たしている証跡になるので、必ず残す。
